# Architectural Decisions

Record significant engineering decisions.

---

## AD-001

Context-First Architecture

Decision

Adopt a centralized Venue Context Engine.

Reason

Avoid duplicate state.

Maintain consistency.

Status

Accepted

---

## AD-002

Gemini as Reasoning Layer

Decision

Gemini generates explanations only.

Business logic remains inside application engines.

Status

Accepted

---

## AD-003

Feature-Based Folder Structure

Decision

Organize modules by feature instead of file type.

Status

Accepted

---

## AD-004

Simulation Before Live APIs

Decision

Build all features against Simulation Engine.

Replace with live integrations later.

Status

Accepted

---

## AD-005

Progressive Web App

Decision

Pulse will be delivered as a PWA instead of a native mobile application.

Reason

- No installation friction
- QR-code friendly
- Cross-platform
- Better suited for temporary event attendees

Status

Accepted
